## Working agreements

- Prefer `bun` to npm, pnpm or yarn always
- Always prefer SPEC driven development for new and existing apps
- Follow `Documentation/InterfaceStandards.md` for UI copy, control naming, and
  visual hierarchy before adding or changing user-facing interface text
- Use clear, concise, technical sentence case for custom Bonsai UI copy,
  including custom menu commands. Capitalize only proper nouns, acronyms, and
  product terms such as Git, GitHub, and LFS.
