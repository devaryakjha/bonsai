# Spec 0082: OSS Contributor Readiness

## Objective

Make the repository easier to open source by documenting the expected local
development, validation, contribution, and security-reporting workflow.

## Requirements

- Add contributor guidance that keeps Bonsai spec-driven and SwiftPM-first.
- Document the validation commands used for code changes.
- Document UI-copy review expectations through the existing interface standards.
- Add security-reporting guidance for a pre-v1 open-source project.
- Link public project docs from the README.

## Acceptance

- `CONTRIBUTING.md` exists and references specs, validation, and UI standards.
- `SECURITY.md` exists and sets a clear supported-version/reporting policy.
- README links to the contributor and security documents.
- Whitespace checks pass.
